# v0.3 Agent Memory Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build v0.3 protocol, retrieval, governance, dashboard, Agent Card, and documentation upgrades for Across Context.

**Architecture:** Keep the current zero-dependency Node.js module boundaries. `ContextVault` remains the storage/lifecycle API, `semantic-search.js` owns retrieval scoring, `mcp.js` owns capability definitions, `mcp-server.js` dispatches JSON-RPC, and `dashboard.js` renders the local governance UI.

**Tech Stack:** Node.js 20 ESM, built-in `node:test`, JSONL local storage, stdio JSON-RPC MCP server, dependency-free server-rendered dashboard.

---

## File Structure

- Modify `src/semantic-search.js`: weighted local retrieval, explanations, empty-query dashboard support.
- Modify `src/vault.js`: search filters, batch status update, recent-memory listing semantics.

- Modify `src/dashboard.js`: search/filter APIs, mutation endpoints, interactive rendered HTML.

- Modify `src/exporters.js`: generated workflow instructions for v0.3.
- Modify `README.md`: English and Chinese v0.3 workflow documentation.
- Add/modify tests under `tests/*.test.js` for each changed behavior.
